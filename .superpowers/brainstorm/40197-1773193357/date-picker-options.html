How should the date range control work?

This controls what data all platforms show. Pick the model that fits how you actually use the dashboard.

A

Preset Chips — Quick & Simple

Fixed buttons: 7D · 30D · 90D · This Month · Last Month · YTD. One click, done. No custom ranges.

7D 30D 90D This Month Last Month YTD

Pros

  • Fastest to use
  • Easiest to build
  • No edge cases

Cons

  • Can't do custom ranges
  • Less flexible
B

Presets + Custom Date Range

Same chips as A, but with a calendar picker for custom start/end dates when you need them.

30D 90D YTD Custom...
📅 Jan 1, 2026 → Feb 28, 2026

Pros

  • Handles 95% of use cases fast
  • Escape hatch for edge cases

Cons

  • Slightly more to build
  • Monthly-only platforms need clear labeling
C

Month Selector — Simple & Honest

Since most platforms are month-granular anyway, just pick a month. Clean, no false precision.

‹ February 2026 ›
Shows all data for the selected month. Compare to previous month with one click.

Pros

  • Matches how platforms report
  • No false precision on daily data
  • Simplest to build correctly

Cons

  • Can't see "last 30 days" across month boundary
