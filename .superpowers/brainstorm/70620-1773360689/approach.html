Content Intelligence — Where should this live?

We're building a deep analysis engine for Nicki's Spring 2025 content. Pick the best fit for how you'd use it.

A

New Page in creator-metrics Dashboard

A "Content Intelligence" tab added to the existing app at creator-metrics.vercel.app. Imports the exported data once, stores it in Supabase, and lives alongside the earnings dashboard permanently. Can be referenced anytime — this year and beyond.

Pros

  • Lives with all the other creator data
  • Reusable for future content cycles
  • Can cross-reference with live LTK earnings automatically

Cons

  • More build time
  • Requires Supabase schema work
B

Standalone Analysis Report

A self-contained interactive HTML report (or Python notebook) that runs the analysis locally from the exported files and outputs a rich visual report — no Supabase, no deployment needed. One-time deep dive you can open in any browser.

Pros

  • Fast to build and run
  • No infra changes
  • Can email/share as a file

Cons

  • Not live/ongoing
  • Can't auto-update with new data
C

Both — Report Now, Dashboard Later

Start with the standalone report immediately using what you've uploaded — fast insights this week. Then, once the Amazon files arrive, promote it into the creator-metrics dashboard as a permanent "Content Lab" section.

Pros

  • Get insights now while data is fresh
  • Build the right thing for the permanent dashboard

Cons

  • Some duplicated work
